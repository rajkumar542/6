Click Counter
useState · onClick · event handler
Code — App.js
import { useState } from "react";

function App() {
  const [count, setCount] = useState(0);

  return (
    <div
      onClick={() => setCount(count+1)}
      style={{
        width:"100%", height:"200px",
        border:"10px dashed #d61b1b"
      }}
    >
      <h1>mmbitan {count}</h1>
      <h1>Inga vandhu mmbu</h1>
    </div>
  );
}

export default App;
Output Preview

mmbitan 0

Inga vandhu mmbu

👆 Click the box to increment

📖 Explanation
What is useState?
State Hook
useState(0) creates a state variable count starting at 0. setCount updates it. Every time state changes, React re-renders the component automatically.
onClick Event
Event Handler
onClick={() => setCount(count+1)} — when the user clicks the div, this arrow function runs and increases count by 1. The => creates an inline function.
JSX Expression
Displaying State
{count} inside JSX renders the current value. Curly braces {} let you embed any JavaScript expression directly into your HTML-like JSX code.
Inline Style
style={{ }} in JSX
Styles in React are JavaScript objects inside double curly braces. CSS property names become camelCase — border-style → borderStyle, background-color → backgroundColor.
How it works — flow
User clicks div
→
onClick fires
→
setCount(count+1)
→
state updates
→
Re-render with new count

Component Props
function component · props · reusable
Code — App.js
import React from "react";

function Message(props) {
  return (
    <h2>
      Hello, {props.name}!
      Welcome to React.
    </h2>
  );
}

function App() {
  return (
    <div style={{textAlign:"center"}}>
      <Message name="A" />
      <Message name="B" />
    </div>
  );
}

export default App;
Output Preview

📖 Explanation
Component
What is Message()?
Message is a custom reusable component — a JavaScript function that returns JSX. Define it once, use it multiple times like an HTML tag: <Message />.
Props
Passing Data to Components
name="A" is a prop (short for property). Props are passed like HTML attributes and received as props.name inside. They let you reuse the same component with different data.
Reusability
One Component, Many Uses
Instead of writing two separate <h2> tags manually, the same Message component is used twice with different name props. Build once, reuse everywhere.
Parent → Child
Component Tree
App is the parent that renders two child Message components. Data flows one-way: parent → child through props. Children cannot change their own props.
How it works — flow
App renders
→
<Message name="A"/>
→
props.name = "A"
→
Hello, A! Welcome to React.

Form + Express Backend
fetch · POST · server.js · data.json
Code — App.js
import { useState } from "react";

function App() {
  const [name, setName] = useState("");

  const submit = async (e) => {
    e.preventDefault();
    await fetch(
      "http://localhost:5000/save",
      {
        method: "POST",
        headers: {
          "Content-Type":"application/json"
        },
        body: JSON.stringify({ name }),
      }
    );
    setName("");
  };

  return (
    <form onSubmit={submit}>
      <input
        value={name}
        onChange={
          (e) => setName(e.target.value)
        }
        placeholder="Enter name"
      />
      <button>Submit</button>
    </form>
  );
}
export default App;

Code — server.js
const express = require("express");
const fs = require("fs");
const path = require("path");
const cors = require("cors");

const app = express();
app.use(cors());
app.use(express.json());

const filePath = path.join(
  __dirname, "data.json"
);

app.post("/save", (req, res) => {
  let data = [];
  if (fs.existsSync(filePath)) {
    data = JSON.parse(
      fs.readFileSync(
        filePath, "utf8"
      ) || "[]"
    );
  }
  data.push(req.body);
  fs.writeFileSync(filePath,
    JSON.stringify(data,null,2));
  res.send("Saved");
});

app.listen(5000, () =>
  console.log("Server started on port 5000"));
📖 Explanation
Controlled Input
value + onChange
value={name} ties the input to React state. onChange={(e)=>setName(e.target.value)} updates state on every keystroke. This is a "controlled component".
async / await
Sending Data to Server
async makes the function asynchronous. await fetch(...) waits for the POST request to finish before continuing. Without await, the code runs before the server responds.
fetch API
HTTP POST Request
fetch("http://localhost:5000/save", { method:"POST", ... }) sends data to Express. JSON.stringify({name}) converts the JS object into a JSON string for the request body.
e.preventDefault()
Prevent Page Reload
By default, submitting a form reloads the page. e.preventDefault() stops this so React handles the submit itself. Without it, the page refreshes and state is lost.
CORS
Why cors is needed
React runs on :3000, server on :5000. Browsers block cross-origin requests by default. cors() middleware in Express allows requests from different ports.
fs module
Reading & Writing Files
Node's built-in fs reads data.json using readFileSync, adds the new name, then saves back with writeFileSync. Like a simple file-based database.
Full request flow
User types name
→
onChange updates state
→
Submit clicked
→
fetch POST to :5000/save
→
Server writes data.json
→
Input cleared

Number Sign Checker
useState · ternary operator · conditional render
Code — App.js
import React, { useState } from "react";

function App() {
  const [num, setNum] = useState("");

  return (
    <div>
      <h2>Enter Number</h2>
      <input
        type="number"
        value={num}
        onChange={
          (e) => setNum(e.target.value)
        }
        placeholder="Enter number"
      />
      {
        num === ""
          ? <p>Enter a number</p>
          : num > 0
          ? <p>Positive</p>
          : num < 0
          ? <p>Negative</p>
          : <p>Zero</p>
      }
    </div>
  );
}
export default App;

Enter Number

📖 Explanation
Ternary Operator
condition ? A : B
Shorthand if-else. Format: condition ? trueValue : falseValue. Here chained — if empty show prompt, else if >0 "Positive", else if <0 "Negative", else "Zero".
Conditional Rendering
Show Different JSX
React renders different elements based on state. The { } block evaluates the ternary and returns the correct <p> tag. React swaps the old element for the new one on each re-render.
useState("")
Initial State is Empty String
State starts as "", not 0. Checking num === "" shows a hint message before the user types anything, instead of immediately showing "Zero" by default.
type="number"
HTML Number Input
type="number" restricts keyboard to numeric entry and shows up/down arrows. The value from e.target.value is always a string — JS's > and < handle type coercion automatically.
Ternary logic — step by step

→
"Enter a number"
|
num > 0 ?
→
"Positive"
|
num < 0 ?
→
"Negative" else "Zero"

Item List
useState · spread operator · .map()
Code — App.js
import React, { useState } from "react";

function App() {
  const [item, setItem] = useState("");
  const [list, setList] = useState([]);

  const addItem = () => {
    setList([...list, item]);
    setItem("");
  };

  return (
    <div>
      <h2>Item List</h2>
      <input
        type="text"
        value={item}
        onChange={
          (e) => setItem(e.target.value)
        }
        placeholder="Enter item"
      />
      <button onClick={addItem}>
        Add
      </button>
      <ul>
        {list.map((val, idx) => (
          <li key={idx}>{val}</li>
        ))}
      </ul>
    </div>
  );
}
export default App;
Output Preview

Item List

    📖 Explanation
    Two States
    item + list
    item holds current text in the input. list holds all added items as an array. Two separate useState calls manage two different pieces of data independently.
    Spread Operator
    [...list, item]
    [...list, item] creates a new array by copying all existing items and appending the new one. Never mutate the old array — React needs a new array reference to detect the change and re-render.
    .map()
    Rendering Lists
    list.map((val, idx) => <li key={idx}>{val}</li>) loops through each item and returns a <li> JSX element. React renders all of them inside the <ul>.
    key prop
    Why key={idx}?
    When rendering lists, React needs a unique key on each element to track which item is which. idx (the array index) is used here. For production, a unique ID is better than index.
    How addItem works
    Type "apple"
    →
    item = "apple"
    →
    Click Add
    →
    setList([...list,"apple"])
    →
    setItem("") clears input
    →
    <li> appears in list

    Image Slider
    useState · dynamic src · picsum.photos API
    Code — App.js
    import React, { useState } from "react";

function App() {
  const [num, setNum] = useState(1);

  function next() {
    setNum(num + 1);
  }

  return (
    <div style={{
      display:"flex",
      flexDirection:"column",
      justifyContent:"center",
      alignItems:"center"
    }}>
      <h2>Image Slider</h2>

      <img
        src={`https://picsum.photos
          /500/500?random=${num}`}
        alt="img"
      />

      <button onClick={next}>
        Next Image
      </button>
    </div>
  );
}
export default App;
    Output Preview

    Image Slider

    📖 Explanation
    Template Literal
    Dynamic Image URL
    `...?random=${num}` uses a backtick template literal to embed num into the URL. Every time num changes, src gets a new URL and the browser loads a different image.
    picsum.photos
    Free Random Image API
    Lorem Picsum is a free public API that returns random 500×500 images. Adding ?random=N with different numbers ensures a unique image each time. No API key needed — great for prototyping.
    useState(1)
    Counter as Image Index
    State starts at 1 and increments on each click. This number is plugged into the API URL. Instead of storing image URLs in an array, the API generates them on demand using the number as a seed.
    Flexbox Layout
    Centering with style
    display:"flex", flexDirection:"column", justifyContent:"center", alignItems:"center" centers everything using CSS Flexbox — written as a JS object in React's inline style prop.
    How the image changes
    num = 1 (initial)
    →
    src = "…/random=1"
    →
    Click "Next Image"
    →
    setNum(2)
    →
    src = "…/random=2"
    →
    New image loads
